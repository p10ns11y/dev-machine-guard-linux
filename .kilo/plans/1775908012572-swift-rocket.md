# DevGuard Scanner Refactoring Plan

## Project Overview
DevGuard Scanner is a lightweight, local security tool for scanning developer environments on Linux/macOS. It detects vulnerable Node.js packages, IDE extensions, and AI tools, with extensible detectors like git history analysis.

## Current Strengths
- Clean, modular Bash architecture
- Local-only (no telemetry/cloud)
- Extensible detector system
- Comprehensive CLI with good help
- Solid testing suite (smoke, unit, integration)
- Proper error handling with set -euo pipefail
- Good separation of concerns

## Areas for Improvement

### Security Enhancements
1. **Input Sanitization**
   - Current package name validation is basic (`^[a-zA-Z0-9@._/-]+$`)
   - Add path traversal protection (`../`, absolute paths)
   - Sanitize detector file paths
   - Validate regex patterns for --version

2. **Command Injection Prevention**
   - All file paths and arguments passed to commands need proper quoting
   - Use arrays for command arguments instead of string concatenation
   - Validate and sanitize search paths

3. **Resource Limits**
   - Add configurable memory/time limits for scans
   - Implement proper cleanup on interruption
   - Add rate limiting for network operations (future)

### Developer Experience (DX) Improvements
1. **File Naming and Structure**
   - Rename main script from `security-dev-machine-guard.sh` to `devguard.sh`
   - Create `lib/` directory for shared functions
   - Move detectors to `detectors/` directory
   - Standardize naming: snake_case for scripts, kebab-case for flags

2. **Configuration**
   - Add proper config file support (.devguardrc with JSON/YAML)
   - Environment variable support
   - Profile-based configurations

3. **CLI Improvements**
   - Consistent flag naming (--package-name, --version-regex, --enable-node)
   - Add --verbose/-v for detailed output
   - Support for config file override flags
   - Better progress indicators for long scans

4. **Installation and Distribution**
   - Add install script for system-wide installation
   - Package as .deb/.rpm for Linux distros
   - Add completion scripts for bash/zsh

### Performance Optimizations
1. **Algorithm Improvements**
   - Batch git operations instead of per-file calls
   - Use parallel processing for independent detectors
   - Implement caching for repeated scans
   - Optimize regex patterns for better performance

2. **Data Structures**
   - Use associative arrays for package tracking
   - Implement efficient deduplication
   - Add result caching with TTL

3. **Execution Flow**
   - Lazy loading of detectors
   - Early exit on critical errors
   - Stream processing for large outputs

### Code Quality and Maintainability
1. **Code Organization**
   - Extract common functions to shared library
   - Implement proper error codes and messages
   - Add comprehensive logging levels
   - Create utility functions for repeated operations

2. **Testing Enhancements**
   - Add performance benchmarks
   - Expand test coverage (edge cases, error conditions)
   - Add property-based testing
   - Implement CI/CD pipeline

3. **Documentation**
   - Add inline documentation for complex functions
   - Create API documentation for detector interface
   - Add troubleshooting guide
   - Document security considerations
